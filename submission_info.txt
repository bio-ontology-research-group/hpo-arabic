## Translation Description (3–5 sentences, mandatory per HPO guidelines)
This Arabic translation of the Human Phenotype Ontology was produced as part of the
PAVS (Phenotypic and Variant Standardization) project, which curates genomic variant
and clinical phenotype data from Saudi Arabian patients with rare genetic diseases.
Translations were generated using GPT-4o (via OpenRouter) with a structured prompt
enforcing domain-specific Arabic medical terminology standards — in particular,
consistent use of شذوذ (structural abnormality), خلل (functional deficit),
اضطراب (complex/systemic disturbance), نقص تنسج (hypoplasia),
and عدم التكون (aplasia/agenesis). All translations are CANDIDATE status
(machine-generated, pending expert review) and are offered to the community
for refinement and eventual promotion to OFFICIAL.

## Submission Instructions

### Step 1 — Join the mailing list (if not already done)
  https://groups.google.com/g/hpo-translations
  Introduce the PAVS Arabic translation and your affiliation.

### Step 2 — Fork the repository
  https://github.com/obophenotype/hpo-translations
  Create a feature branch, e.g.: git checkout -b add-arabic-pavs-update

### Step 3 — Deposit translation files
  The target directory is:
    babelon/hp-ar.babelon.tsv      ← primary submission file
  An existing hp-ar.babelon.tsv already exists in the repo.
  Strategy options:
    a) REPLACE: overwrite with the new file (our version covers 19,408 terms;
       review the existing file first to avoid losing any OFFICIAL-status rows).
    b) MERGE: combine rows from the existing file, keeping any OFFICIAL rows and
       replacing/adding CANDIDATE rows from our file. Recommended if the existing
       file has human-reviewed entries.

  For synonyms:
    babelon/hp-ar.synonyms.tsv     ← deposit hp-ar-synonyms.robot.tsv here
                                     (rename to match the repo convention)

### Step 4 — Open a Pull Request
  Title: "Update Arabic (ar) translation — PAVS pipeline, 19,408 terms"
  Body should include:
    - Submitter name & ORCID
    - Organization & ROR
    - Description (copy from above)
    - Coverage statistics
    - Note that status is CANDIDATE and expert review is welcome
    - Link to PAVS project (https://github.com/leechuck/pavs) for context

  Target branch: main

### Step 5 — Alternative: open an issue
  If a direct PR is not accepted for bulk machine translations, open a GitHub issue at:
    https://github.com/obophenotype/hpo-translations/issues
  Attach or link the babelon TSV and request the maintainers to merge it.

### Step 6 — Alternative: email submission
  If unfamiliar with GitHub or if the maintainers prefer it, email the files directly.
  Find HPOIE contacts at: https://obophenotype.github.io/hpo-translations/submit/
  Use subject: "Arabic HPO translation submission — PAVS/KAUST"
